Transitioning to ARC Release Notes－翻译

ARC Rules
1.不能显示调用dealloc，retain，release，retainCount，autorelease。延伸的，不能调用@selector(retain),@selector(release)等等。
因为你不能去主动的释放实例。
在ARC中dealloc方法不需要调用［super dealloc］,这个调用是compiler自动调用的。
可以继续使用CFRetain，CFRelease和其他相关的Core Foundation－style 对象。

2.不能使用NSAllocateObject或者NSDeallocateObject
你应该使用alloc来创造对象，而deallocate由runtime来处理。

3.不能在C structures里使用对象指针
尽量使用Objective-C对象

4.不支持id和void之间的自动转换
当需要id和void之间转换时（Objective-C对象和Core Foundation类型），你需要明确的声明。详情见Managing Toll-Free Bridging

5.不能使用NSAutoreleasePool对象
用@autoreleasepool block代替，比NSAutoreleasePool效率更高

6.不能使用NSZone
NSZone已经被新的Objective－C runtime自动忽略

7.不能用new开头来声明属性
http://blog.csdn.net/meegomeego/article/details/17632515
若是用new开头，那么编译器认为gettre方法是生成一个新的对象
// Won’t work:
@property NSString *newTitle;

// Works:
@property (getter=theNewTitle) NSString *newTitle;

Property Attributes
strong 相当于 retain
weak 相当于assign，当实例被释放后，会自动赋nil（而不再是野指针）
在ARC下，默认strong

Variable Qualifiers
strong 默认，保证对象是alive的 weak 引用，并不保证对象alive，当没有strong的引用时，会自动设nil
unsafe_unretained 同weak，不同的是，当没有strong引用时，不会设nil，而是悬空指针
__autoreleasing 用在函数传参，且这个参数在函数返回之后会释放时。http://blog.txx.im/blog/2013/12/04/about-autoreleasing/

正确的写法
MyClass * __weak myWeakReference;

使用weak的时候需要注意。
1.这种情况下，声明之后就立即被释放了
NSString * weak string = [[NSString alloc] initWithFormat:@”First name: %@”,[self firstName]];
NSLog(@”string: %@”,string);
打印结果 string: (null)

2.当你将一个strong变量做为函数autoreleasing参数传递时，编译器会自动生成一个临时变量。解决方法是，将变量声明为autoreleasing；或将函数参数定义为id strong
-(BOOL)performOperationWithError:(NSError * autoreleasing )error;
/////////////////////
NSError error;
BOOL OK = [myObject performOperationWithError:&error];
if (!OK) {
// Report the error.
// …
/////////////////////

转译为

//////////////////////
NSError __strong error;
NSError __autoreleasing tmp = error;
BOOL OK = [myObject performOperationWithError:&tmp];
error = tmp;
if (!OK) {
// Report the error.
// …
//////////////////////

Use Lifetime Qualifiers to Avoid Strong Reference Cycles
在MRC中 block id x; 不持有x
而在ARC中，block id x; 默认持有x。 若需要有MRC的效果需这样声明unsafe_unretained block id x;更好的方法是使用__weak或置nil来打破循环引用。

ARC Uses a New Statement to Manage Autorelease Pools
在ARC中，应当这样写
@autoreleasepool {

// Code, such as a loop that creates a large number of temporary objects.

}
在入口处建立一个自动释放池，并入栈。
在出口处（break,return,goto,fall-through,等等）自动释放池出栈
当出现异常退出时，自动释放池不会出栈。
@autoreleasepool 的效率比NSAutoreleasePool高6倍

Patterns for Managing Outlets Become Consistent Across Platforms
outlets should be weak, except for those from File’s Owner to top-level objects in a nib file (or a storyboard scene) which should be strong.

Stack Variables Are Initialized with nil
在ARC中，strong，weak，autoreleasing 堆变量会被自动初始化为nil

Use Compiler Flags to Enable and Disable ARC
-fobjc-arc
-fno-objc-arc

Managing Toll-Free Bridging
用的很少，忽略

Oracle RAC

RAC五大特点

  1. 双机并行
    双机热备环境中，始终有一台机器作为备用机，只有当主节点出问题的时候才会切换到备用机上；如果主机一直没有问题，备用机始终处于空闲状态。同时，主机切到备机需要消耗时间。
    RAC环境中，两个节点的集群节点间是一种并行运行的关系，充分利用服务器资源。机器间切换快速，针对存在的绘画之需要数十秒的时间就可以完成失败切换过程，对新回话的创建没有影响。
  2. 高可用性
    高可用性包括两个部分：确保数据不丢失；确保不停机。
    RAC中包含了非常多的高可用特性，主要包含如下几点：
    　　·实现节点间的负载均衡。
    　　·实现失败切换的功能。
    　　·通过Service组件来控制客户端的访问路径。
    　　·集群软件能够自动化管理各个资源，并且有定时的节点状态检测机制，能自动对一些失败的进程以及心跳检测失败的节点进行重启，使其重新恢复到正常的运行状态。
  3. 易伸缩性
    当系统需要更多的处理能力或者需要增加存储时，通过添加另一台服务器或存储设备到集群中，能够在不停机的情况下获得水平的扩展。
    在一个集群中， Clusterware和RAC支持多达100个集群节点。
    11gR2版本中推出了网格即插即用(Grid Plug and Play，GPnP)，可以实现节点的快速添加。
  4. 低成本
    通过多台普通的PC服务器组成一个集群，可以提高集群的处理能力，这样要比采用一台高性能的服务器的成本低很多。
  5. 高吞吐量
    在RAC的架构中，多个实例分布在多个服务器上，能同时打开同一个数据库，而每个实例能够接收相等数量的客户端请求，这样，随着服务器的增加，吞吐量也在不断地增加。

RAC存在的问题：

  1. 稳定性
  2. 高性能
    在目前普遍使用千兆网络的硬件环境中，很多时候系统的数据库从原来的单机迁移至RAC环境，系统的性能反而下降。在这种情况下，数据库管理员应该根据RAC的特点对系统调整提出合理的建议，经过合理的设计、开发，使用RAC是能够提高系统的处理性能的。

FPS追踪工具

1. DXFPSLabel

原理：
1.在[NSRunLoop currentRunLoop]的UITrackingRunLoopMode中添加事件_scrollingStatusDidChange
2.1手势拖动
2.2调起_scrollingStatusDidChange
2.3初始化_displayLink，添加事件_screenDidUpdateWhileScrolling，添加进[NSRunLoop currentRunLoop]的UITrackingRunLoopMode模式下。
2.4初始化
_framesInLastInterval/_lastLogTime/_displayLink 启动
3.1手势停止
3.2调起_scrollingStatusDidChange
3.3_displayLink 暂停
3.4在[NSRunLoop currentRunLoop]的UITrackingRunLoopMode中添加事件_scrollingStatusDidChange

问题：
1.[NSRunLoop currentRunLoop]
在[NSRunLoop currentRunLoop]中添加事件，能够监听所有页面的事件。

2.CADisplayLink
CADisplayLink是一个定时器对象，用来同步页面的刷新。创建一个display link，提供target和selector，用来在屏幕刷新时调用。然后将这个display link添加到一个run loop。这些设置完成后，屏幕的内容需要刷新的时候会调用selector。target可以读取display link的时间戳timestamp属性来取回前一帧的时间。举例，用来播放电影的app可以用时间戳来计算接下来播放哪一帧。app可以根据时间戳来判断当前动画对象在下一帧如何展示。duration给出两个frames之间的时间。你可以根据这个值来计算帧率，下一帧的合适时间，调整绘制动作让下一帧在合适的时间出现。
app可以设置paused属性为YES来停止通知。当app无法在下一个时间点提供合适的帧时，可以降低帧率。提供缓慢但连续的帧率会比跳帧有更高的界面平滑度。你可以修改frameInterval属性来提高两帧之间的时间（降低帧率）。
当app停止使用display link时，可以调用invalidate来将其在run loops中移除，且接触其与target之间的关联。
CADisplayLink无法被继承。
iOS 3.1 and later

牛蒡茶

首选来自“中国牛蒡之乡”—苍山

最后看冲泡出来的效果，一般好的牛蒡茶开水冲泡后有一股浓郁的人参香气，茶色红润，味道甘醇，耐冲泡。反之，一般的牛蒡茶就相差甚远。好的牛蒡茶见效，经常饮用会有口干、腹胀、放屁、排便功能增强、肚饿等类似的好转反应。而普通的则要差很多，而且很多达不到卫生、质量标准的牛蒡茶还会导致其他疾病。

如何选购牛蒡茶
⒈原料：原料直接决定了牛蒡茶的品质，选用来自“牛蒡之乡”苍山的牛蒡是最好的，因为那里有特别适合牛蒡生长的土壤以及新鲜干净的空气、水分，所以培育出的牛蒡营养丰富、生命力强，是中国牛蒡的源产地和出口首选地；
⒉外观：看茶片是否是非空心（茶片中心呈白色空洞状）（空心的牛蒡片一般是低端的），看是否为小圆片（小圆片是优质黄金牛蒡），茶水色泽是否呈红润的酒红色且无杂质；闻气味是否香浓。好的牛蒡茶开水冲泡后会有一股浓郁的人参或者咖啡的香味，且茶色红润；
⒊口感：好的牛蒡茶味道甘醇柔滑，耐冲泡，饮后口有余香、淡淡回甘。反之，一般的牛蒡茶就相差甚远；
⒋包装：包装不仅仅是体现牛蒡茶的档次，更是质量、效果的保证。看外包装是否有食品安全认证QS标志；看内包装是否严密，内包装最好采用铝膜纸，其保鲜和防水性极好。

注意事项
编辑
饮用时应该注意的事项，牛蒡茶在饮用时不宜过浓，过浓容易引起上火，就像我们平时喝高汤一样，喝的量多，就容易上火，在泡制的时候可以适当的加入一些蜂蜜这样就不会上火了！
还有牛蒡茶就有排毒养颜，减肥的功效，女性在饮用时要注意月经期最好不要饮用！
每天5-8克，便秘者、减肥者可酌情增加饮用牛蒡茶。
牛蒡茶不能冷饮，冷饮会导致腹泻。

喝牛蒡茶却有些注意事项。
⒈初饮者可能出现排便功能增强（不是水泻）现象。
2、因牛蒡茶有助于辅助稳定降低高血压，建议低血压患者间歇饮用。
3、不与其它茶混饮，泡发的茶片最后食掉最佳。
4、坚持饮用一个月就会有明显效果。

牛蒡茶色泽金黄、香味宜人，是老幼四季皆宜的保健饮品，也可炖、煮、涮、做汤等。[1] 牛蒡茶具有通经络活血化淤的作用早在李时珍的《本草纲目》中就有记载牛蒡能“通十二经脉，除五脏恶气”，能调整女性经期，对于女性的气滞血淤的症状和久坐办公室的上班族有很好作用。.牛蒡茶不宜过浓，过浓容易引起上火，在泡 制的时候可以适当的加入一些蜂蜜这样就不会上火了![3

具体做法：
热饮：每次5-8克（备注7到8片），便秘者、减肥者可酌情增加。
用开水冲泡2-3分钟即可饮用（可反复冲泡多次），也可以配以冰糖、红枣、洋参冲泡效果更佳，还可以放少许蜂蜜，预防上火！

营养价值：

牛蒡含菊糖、纤维素、蛋白质、钙、磷、铁等人体所需的多种维生素及矿物质，其中胡萝卜素含量比胡萝卜高150倍，蛋白质和钙的含量为根茎类之首。牛蒡根含有菊糖（人体内不含菊糖）及挥发油、牛蒡酸、多种多酚物质及醛类，并富含纤维素和氨基酸。
牛蒡根含有人体必需的各种氨基酸，且含量较高，尤其是具有特殊药理作用的氨基酸含量高，如具有健脑作用的天门冬氨酸占总氨基酸的25%-28%，精氨酸占18%-20%，且含有Ca、Mg、Fe、 Mn、 Zn等人体必需的宏量元素和微量元素。[6-7]

药用功效：
治疗糖尿病、高血压、高血脂、抗癌等作用。可降血糖、降血压、血脂、治疗失眠，提高人体免疫力等功效。
牛蒡全植物含有抗菌成分，其中叶含抗菌成分最多，主要抗金黄色葡萄球菌。
抗衰老和清除氧自由基作用
人类生命在正常活动代谢过程中，会产生一种有害于身体健康，促使细胞衰老的物质——氧自由基。它们能够促使产生脂褐斑色素——老年斑的生成和堆积。老年斑在体表的出现，表示机体中细胞已进入衰老阶段。牛旁根中含有过氧化物酶，它能增强细胞免疫机制的活力，清除体内氧自由基，阻止脂褐质色素在体内的生成和堆积，抗衰防老，为机体提供了对抗和清除氧自由基的内护环境。
抗癌作用
牛蒡苦素能抑制癌细胞中磷酸果糖基酶的活性，牛蒡甙元也有抗癌活性，同时牛蒡子甙元还具有抗老年性痴呆作用。现代科学研究结果认为，牛蒡含黄酮甙类化合物，对恶性肿瘤具有一定抗性，其粗提取物呈选择毒性，较低量就可以抑制癌细胞增殖，使肿瘤细胞向正常细胞接近。[9]
移除废水中重金属作用
牛蒡对金属离子的吸附能力依次为：Pb>Cd>Hg>Ca>Zn。
其它作用
牛蒡甙和牛蒡酚有抗肾炎活性
，能有效地治疗急性进行性肾炎和慢性肾小球肾，做肾病治疗剂。牛蒡的主要成分木酚素具有抑制血小板活化因子对血小板结合作用，可以做血小板活化因子拮抗药。[10]
